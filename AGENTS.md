## Critical rules

- **Security and Privacy**: Since this is a public project, strictly ensure that personal local development environment settings (absolute paths, local folder names, etc.), security settings (API keys, tokens, etc.), or sensitive personal information are not hard-coded or committed in the code and documentation.
- **Agent-facing Comments and Documentation Guidelines**: All comments are for providing context to the agent, not for the developer (User), and must be written exclusively in English. Non-English comments and code are not allowed. Do not write supplementary explanations or documentation for things that are self-evident from the code or unnecessary for agent tasks.
- Assume that the storage structure and rename method may have changed after upgrading the agent CLI.
- External CLI rename paths are only trusted when actual storage file modifications are verified.
- If you change the rename/session-title code, do not end with just `cargo test -q`; you must also perform manual CLI verification.
- Update the code and documentation simultaneously when the storage structure changes.
- If the usage parsing code has been modified or the agent CLI has been upgraded, use `--usage-probe` to cross-verify against the actual CLI screen (be careful not to misinterpret absolute times vs. countdowns).
- If the model list parsing code has been modified or the agent CLI has been upgraded, use `--model-probe` to cross-verify against the actual CLI (`/model`, `codex debug models`, `agy models`). Since the CLI does not filter out invalid model names (agy quietly falls back to the default model), s7s is responsible for list accuracy.
- If parsing related to rewind/backtrack (claude `parentUuid` branch / codex `thread_rolled_back`) is modified or the agent CLI is upgraded, perform a rewind in the actual CLI to cross-verify the saved file diff with the s7s preview (refer to iteration 39). Since agy destructively rewrites the storage, it is normal that it has no parser handling.
- If the turn selection logic of the session context parser (`src/session_context/`) or the list parser is changed, verify real-data turn parity (List Q count == Detail == CLI turn count) using `cargo test real_data_turn_parity -- --ignored --nocapture`. The initial prompt injection method should be re-verified upon agent CLI upgrade (claude/codex positional, agy `--prompt-interactive`).
- After changing the layout and UI of the New Session dialog, you must run `cargo build --release`.
- After completing code modifications, you must run `cargo build --release` to confirm the build succeeds.
- All files to be committed to the repository (code, documentation, etc.) and git commit messages must be written in English. Any files containing Korean must be added to .gitignore after obtaining the user's permission.
